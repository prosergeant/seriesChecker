# page.tsx

Главная страница приложения (`/`). Содержит весь основной UI: поиск, список прогресса, карточки сериалов.

## Компоненты

### `SearchResults`
Выпадающий список результатов поиска. Появляется при вводе >=2 символов. Запросы через TanStack Query с ключом `["search", query]`.

### `ProgressCard`
Карточка одного сериала/фильма из списка пользователя. Показывает постер, название, статус, кнопки действий.

**Ключ карточки:** `item.series_id` (= `kinopoisk_id`) — важно для стабильности стейта при изменении списка (чтобы модалки и редактирование не сбрасывались при добавлении новых элементов). Раньше использовался `item.id`, но бекенд не заполняет это поле в `GetListByStatus` — у всех элементов `id: 0`, что вызывало дублирование ключей и ремаунт компонентов.

**Кнопки:**
- Редактирование сезон/серия (только для сериалов) — при сохранении **автоматически ставит статус "watching"**
- Статус-бейдж — кликабельный `DropdownMenu` для смены статуса (watching/completed/planned/dropped/on_hold)
- "Смотреть" — открывает sspoisk.ru
- `RelatedMoviesModal` — похожие и связанные
- Удалить

### `HomeContent`
Основной layout страницы: header с email/logout, поиск с автодополнением, фильтры по статусу, сетка карточек.

## Изменения (2026-03-21)
- `handleSave` теперь всегда устанавливает `status: "watching"` при сохранении сезона/серии
- Ключ `ProgressCard` изменён с `progress-${series_id}-${index}` на `item.id` (ошибочно), затем на `item.series_id` — финальный фикс: бекенд не заполняет `id` в `GetListByStatus`, поэтому у всех карточек был ключ `0` → дубликаты → ремаунт → сброс модалки
